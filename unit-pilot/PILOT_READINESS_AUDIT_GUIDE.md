# Unit Pilot Readiness Audit

## When to Run

Run the audit on every facilitator device before learners arrive and again after installing or updating the PWA.

## Checks

The audit checks:

1. Core training views are present.
2. Interactive form-practice areas are present.
3. All seven mission codes are present.
4. Run-of-show, assignment board, and observation rubric are mounted.
5. The signed balance field permits shortages.
6. Local browser storage works.
7. The browser supports service workers.
8. The mobile viewport is configured.
9. Every named team has a mission and unnecessary duplicates are absent.
10. Started facilitator observations include all five behaviors.
11. Hosted PWA assets respond successfully.

## Interpretation

- Pass: the check is ready.
- Warning: expected before some event steps, but the facilitator must review it.
- Fail: correct before using the device for training.

File-mode use cannot verify hosted asset delivery. Run the hosted GitHub Pages version for the final audit.

The audit does not change learner progress, unlock content, reveal answers, or transmit data.
